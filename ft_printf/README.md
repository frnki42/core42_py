# ft_printf

## Description

`FT_PRINTF` is a custom `variadic` C function that reimplements the original `printf` function in a much simpler way.
`libftprintf.a` is a combination of my previous library `libft` and this `ft_printf` function with all its dependencies.

The function prints the passed `string` and replaces all given `specifiers` (`%`-sign in combination with another character inside the string), with their respective equivalent from the `va_list` named `lst`.

If there is no `fmt` (abbreviation for `format`) string given, the function returns `-1` immediately.
`fmt` and any additional arguments get stored in a `va_list` typed variable called `lst`.

The function prints any other character than the specifiers as is and converts any given specifier to the value stored in `lst`.
For selecting the correct function depending on the given specifier the function uses a `static` dispatcher helper function.

## Algorithm

Due to the need to handle different `data types`, multiple functions had to be used. Depending on the `data type` different functions will get called inside `print_spec()` which is the `dispatcher`.

The `dispatcher` consists of 5 different functions:

**pf_putchar():** prints the passed integer as `character` to `stdout` and returns `-1` on failure or `1` on success.

**pf_putstr():** prints the passed `string` to stdout and returns its length on success. Returns `(null)` and a size of 6 when a `NULL` pointer is passed.

**pf_putnbr():** prints the passed number (the function can handle numbers up to `LLONG_MAX `and down to `LLONG_MIN + 1`) in its given base. Due to its design the function can be used for the specifiers `d` `i` `u` `x` and `X` for less duplication and a cleaner code base.
**pf_putptr():** prints the passed `pointer address`. uses `uintptr_t` internally for safe pointer-to-integer conversion.

**pf_putinvalid():** prints `invalid specifiers` and the associated `%`-sign.

## Data Structure


`va_list` is a variable type suitable for holding the information needed by the macros:
`va_start`: enables access to the variable arguments.
`va_arg`: expands to an expression type that corresponds to the next parameter from the `va_list`.
`va_end`: performs cleanup for `va_list`

## Instructions

### Compilation

Clone the repository and compile the library using `make`:

```
git clone <repository-url> ft_printf
cd ft_printf
make
```

This produces `libft/libft.a`, a static library archive which is then merged with the `ft_printf` object files to produce `libftprintf.a` in the root directory.

The following Makefile rules are available:

- `make` or `make all` — compiles the library
- `make clean` — removes object files
- `make fclean` — removes object files and the library archive
- `make re` — performs a full recompilation

### Usage

To use `ft_printf` in your project, include the header and link the library:

```c
#include "ft_printf.h"
```

Compile with:

```
cc your_file.c -L/path/to/ft_printf -lftprintf -I/path/to/ft_printf
```
